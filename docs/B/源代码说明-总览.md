# B模块源代码说明文档：总览

## 1. 模块名称

分流策略与推荐算法模块。

## 2. 模块位置

后端源码位置：

```text
backend/src/main/java/com/bjtu/dining/recommendation/
```

测试源码位置：

```text
backend/src/test/java/com/bjtu/dining/recommendation/
```

相关文档位置：

```text
docs/B/
```

## 3. 模块职责

B 模块负责基于 A 模块生成的仿真结果，完成推荐、分流和策略对比计算。该模块不负责生成原始人流数据，也不负责前端图表展示。

当前已实现的功能包括：

- 餐厅、窗口、菜品推荐。
- 已生成推荐结果查询。
- 拥挤窗口分流建议。
- 两个仿真场景的指标对比。
- 中文推荐理由、分流原因和场景对比结论生成。

## 4. 对外接口

接口统一使用 `/api/v1` 前缀。

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/v1/recommendations/generate` | POST | 根据 `runId`、时间点和用户画像生成推荐结果 |
| `/api/v1/recommendations/runs/{runId}` | GET | 查询已生成的推荐结果 |
| `/api/v1/recommendations/diversion` | POST | 根据仿真快照生成分流建议 |
| `/api/v1/strategies/compare` | POST | 对比两个仿真场景的 metrics 指标 |

## 5. 代码目录结构

```text
recommendation/
├── controller/    接口控制层，负责接收 HTTP 请求并返回统一响应
├── dto/           请求和响应数据对象
├── model/         B 模块内部使用的仿真、参数和指标数据模型
├── repository/    CSV 种子数据读取与内存推荐结果缓存
└── service/       推荐、分流、策略对比等核心业务逻辑
```

## 6. 当前数据来源

当前 B 模块尚未接入 A 模块真实仿真接口，暂时使用：

```text
MockSimulationProvider
```

该类会根据 `runId` 生成可复现的内存仿真结果，便于 B 模块独立开发和测试。后续 A 模块接口完成后，应替换为真实仿真结果查询适配器。

CSV 参数数据来自：

```text

data/task-a/dishes.csv
```

## 8. 后续联调说明

后续与 A 模块联调时，建议新增统一接口，例如 `SimulationProvider`，让当前 `MockSimulationProvider` 和未来真实 A 接口适配器共同实现该接口。这样可以保留 mock 测试能力，同时平滑切换到真实仿真数据。
